**********************************************************
* Fichiers publics sont des symlinks vers le code source
**********************************************************

Les fichiers liés sur la page d'accueil README.md sont des symlinks
créés sur Windows avec Command Prompt et la commande suivante :

mklink /H nomDuLienÀCréer CheminAuFichierSource

Le but est d'assurer qu'une mise à jour dans le code source soit
automatiquement reflété dans le document lié et éliminer l'erreur 
d'avoir des versions différentes dans le code source et sur le site
public.

> Par contre, il faut cliquer sur les liens une fois pour enclencher
> une mise à jour du contenu associé au symlink...

******************************
* Sauf les archives zip
******************************

L'exception sont les archives .zip qui n'existent pas dans le code
source et qui sont créées manuellement. Dans leur cas, la date de
création du fichier .zip est incluse dans son nom afin de permettre
la comparaison avec la date de modification des fichiers de code source.
